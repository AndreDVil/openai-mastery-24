# üß± Global Standards

All projects must follow:

### ‚úî Documentation Standards  
- English only  
- 8-section README template  
- clear architecture and examples  
- reproducible instructions  

### ‚úî Demo Guidelines  
- at least one demo per project  
- notebook / CLI / screenshots / video  
- stored in the `/demo` directory  

### ‚úî Git Workflow  
- `main` always stable  
- one branch per project (`projectXX`)  
- feature branches optional  
- mandatory PR with template  
- delete branches after merge  

### ‚úî Definition of Done  
A project is only **DONE** when:  
- README completed  
- skills-report completed  
- demo provided  
- PR approved and merged  
- CERTIFICATE.md updated  

See: **docs/definition-of-done.md**
