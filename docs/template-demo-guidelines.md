# Diretrizes para Demos – openai-mastery-24

Cada projeto deve ter pelo menos **um** dos formatos de demo:

- Notebook executável
- Script CLI com exemplo de uso
- App simples (ex: Streamlit, FastAPI + Swagger)
- Vídeo curto (30–90s) mostrando o sistema em uso
- Capturas de tela com explicação

## Recomendações

- Colocar os artefatos na pasta `demo/` do projeto.
- Nomear arquivos de forma clara:
  - `demo-notebook.ipynb`
  - `demo-cli.gif`
  - `demo-screenshots.md`
  - `demo-video-link.md`
- No `README.md` do projeto, sempre ter uma seção **"Demonstração"** com links para esses artefatos.
